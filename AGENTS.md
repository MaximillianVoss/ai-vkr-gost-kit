# Инструкции Для ИИ-Агентов

При работе с пояснительными записками, ВКР и дипломными проектами используй этот репозиторий как основной набор правил оформления.

Базовые правила:
- сначала прочитай `README.md` и `ai-client-vkr-gost-spec.json`;
- если нужны схемы, дополнительно прочитай `tools/schemegen/README.md`;
- используй ГОСТ-стили из пользовательского шаблона Word как основу оформления;
- если пользователь пишет только `ВКР`, не выбирай профиль автоматически: сначала уточни, это СПО/дипломный проект или ВО/магистратура;
- не выдумывай ФИО, вуз, кафедру, группу, город, год и другие реквизиты; при нехватке данных оставляй плейсхолдеры или запрашивай значения у пользователя.

Правила оформления:
- для `ГОСТ Заголовок 1/2/3` используй размеры `20/18/16 pt` и интервалы `6 pt` до и после, если методичка или шаблон не требуют иного;
- `РЕФЕРАТ`, `ВВЕДЕНИЕ`, `ЗАКЛЮЧЕНИЕ`, `СПИСОК ИСПОЛЬЗОВАННЫХ ИСТОЧНИКОВ` и `ПРИЛОЖЕНИЕ` должны быть без нумерации и оформляться как заголовки главы;
- титульный лист и содержание должны быть в отдельных секциях: они входят в общий счет страниц, но номер на них не печатается;
- видимая нумерация страниц должна начинаться с первого разрешенного структурного элемента после содержания;
- при экспорте PDF проверь, что в оглавлении нет ошибок Word вида `Ошибка! Закладка не определена.`.

Инструментальный порядок:
- основной CLI: `word_agent_toolkit.py`;
- для схем и диаграмм: `tools/schemegen`;
- сначала применяй стили шаблона и встроенную постобработку утилиты;
- VBA-макросы запускай только явно через `--macro`, если они действительно нужны;
- для проверки шаблона используй `inspect-template`, для финальной нормализации `finalize-doc`, для тестовых прогонов `generate-sample-vkrs`.
- если пользователь просит блок-схемы, `IDEF0`, `IDEF1X`, `IDEF3` или `UML`, не рисуй их вручную в Word: сначала собери JSON и отрендери `SVG` через `tools/schemegen`.
- для `IDEF0` соблюдай правило: контекстная диаграмма содержит один блок, дочерняя диаграмма обычно содержит `3..6` блоков, а граничные интерфейсы родителя должны сохраняться на дочерней диаграмме.
- промежуточные JSON и SVG схем разумно складывать в `tmp/diagrams/` или в рабочую папку пользователя, а не смешивать с финальными DOCX без необходимости.

Работа с репозиторием:
- не коммить локальные тестовые выгрузки `test/` и `test_dotm/`, если пользователь отдельно этого не просил;
- не коммить локальные рендеры схем из `tools/schemegen/rendered/`, если пользователь отдельно этого не просил;
- при обновлении правил синхронизируй `README.md`, `ai-client-vkr-gost-spec.json` и `AGENT_PROMPTS_RU.md`, чтобы агентские инструкции не расходились с кодом.
