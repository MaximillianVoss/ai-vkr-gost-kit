# Готовые Prompt-шаблоны

Ниже приведены шаблоны, которые можно отправлять другому ИИ-агенту вместе со ссылкой на репозиторий:

`https://github.com/MaximillianVoss/ai-vkr-gost-kit`

## 1. Короткий Prompt

Используй репозиторий `ai-vkr-gost-kit` как основной источник правил для подготовки и оформления ВКР или дипломного проекта по ГОСТ. Сначала прочитай `README.md` и `ai-client-vkr-gost-spec.json`, затем определи профиль методических указаний: для дипломного проекта / ВКР СПО используй профиль `diploma_project_spo`, для ВКР магистратуры / ВО используй профиль `masters_vkr_vo`. Если пользователь пишет только `ВКР`, не выбирай профиль автоматически и сначала уточни уровень образования. Не подставляй вымышленные ФИО, название вуза, кафедры, города, группы и другие реквизиты: если данных нет, запроси их перед финальной генерацией документа, а в универсальных шаблонах оставляй плейсхолдеры в квадратных скобках. Не смешивай требования разных методичек в одном документе без явного указания пользователя. Если в работе нужны схемы, используй встроенный инструмент `tools/schemegen`, а не рисуй их вручную.

## 2. Основной Prompt

Ты готовишь выпускную квалификационную работу, дипломный проект или пояснительную записку по пользовательскому шаблону Word и локальным методическим указаниям.

Используй репозиторий:
`https://github.com/MaximillianVoss/ai-vkr-gost-kit`

Порядок работы:

1. Сначала прочитай `README.md` и `ai-client-vkr-gost-spec.json`.
2. Определи профиль оформления по типу работы и уровню образования:
   - `diploma_project_spo` для дипломного проекта или ВКР СПО;
   - `masters_vkr_vo` для ВКР магистратуры или другой ВКР высшего образования.
3. Если пользователь написал только `ВКР`, не выбирай профиль автоматически. Сначала уточни, идет ли речь о СПО или о ВО / магистратуре.
4. Если пользователь дал несколько методичек, выбери ту, которая соответствует типу работы и уровню образования, либо запроси приоритет.
5. Используй только подтверждаемые факты из предоставленных материалов. Не выдумывай архитектуру, функции, результаты, тесты, API, реквизиты титульного листа и сопроводительных документов.
6. Для титульного листа, аннотации, задания, рецензии и иных сопроводительных документов:
   - не подставляй реальные ФИО, название вуза, кафедры, института, группы, города или года без данных от пользователя;
   - если документ создается как универсальный шаблон, сохраняй плейсхолдеры вида `[НАЗВАНИЕ ВУЗА]`, `[ФИО АВТОРА]`, `[ФИО РУКОВОДИТЕЛЯ]`, `[ГРУППА]`, `[ГОРОД]`, `[ГОД]`;
   - если создается финальный документ на сдачу, сначала запроси недостающие реквизиты.
7. Не смешивай структуру дипломного проекта СПО и ВКР магистратуры в одном документе без явного разрешения пользователя.
8. Соблюдай правила оформления из выбранной методички и пользовательского `.dotx`- или `.dotm`-шаблона. Если требования конфликтуют, используй стиль шаблона как основу, но доводи итоговый вид до требований методички.
9. Если в работе нужны блок-схемы, `IDEF0`, `IDEF1X`, `IDEF3` или `UML`, сначала собери их через `tools/schemegen` в виде JSON и SVG, затем вставляй в документ как рисунки.
10. Перед финальной выдачей проверь оглавление, нумерацию страниц, подписи рисунков и таблиц, приложения и соответствие структуры выбранному профилю.
11. Если в шаблоне есть полезные макросы Word, можно использовать их через CLI, но при отсутствии макросов или при нежелательности VBA нужно применять встроенную постобработку утилиты для нормализации текста, таблиц, рисунков и специальных заголовков.
12. Для ГОСТ-заголовков используй размеры `20/18/16 pt` для уровней `1/2/3` и интервалы `6 pt` до и после заголовка, если методичка или шаблон не требуют иного.
13. `РЕФЕРАТ`, `ВВЕДЕНИЕ`, `ЗАКЛЮЧЕНИЕ`, `СПИСОК ИСПОЛЬЗОВАННЫХ ИСТОЧНИКОВ` и `ПРИЛОЖЕНИЕ` оформляй как заголовки главы без нумерации; регистр должен совпадать с регистром заголовка главы текущего профиля.
14. Титульный лист и содержание должны жить в отдельных секциях: они входят в общий счет страниц, но номер на них не печатается; видимая нумерация начинается с первого разрешенного структурного элемента после содержания.
15. При экспорте PDF проверь, что оглавление не содержит ошибок Word вида `Ошибка! Закладка не определена.`.

## 3. Prompt Для Агента С Инструментами

Если у тебя есть доступ к Python и Word COM-автоматизации на Windows, используй `word_agent_toolkit.py` из репозитория. Для схем используй `tools/schemegen`.

Рабочий порядок:

1. Выполни `resolve-profile` и определи профиль.
2. Изучи `ai-client-vkr-gost-spec.json`.
3. Если не хватает реквизитов для финального документа, запроси у пользователя:
   - тип работы;
   - уровень образования;
   - название вуза / колледжа;
   - институт / факультет;
   - кафедру / отделение;
   - ФИО автора;
   - ФИО руководителя;
   - группу;
   - город;
   - год.
4. Если задача состоит в создании универсального профиля для другого агента, не заменяй плейсхолдеры реальными значениями.
5. Если задача состоит в создании финального документа, заменяй плейсхолдеры только после получения данных от пользователя.
6. Если нужны диаграммы, перейди в `tools/schemegen`, проверь JSON через `validate`, отрендери SVG через `render` или `render-all`, а затем вставь результат в документ.

## 4. Шаблон Пользовательского Сообщения

Ниже шаблон, который можно отправлять вместе со ссылкой на репозиторий:

```text
Используй этот репозиторий как базовый набор правил оформления:
https://github.com/MaximillianVoss/ai-vkr-gost-kit

Нужно подготовить [ТИП РАБОТЫ] по теме: [ТЕМА].

Методические указания:
- [ФАЙЛ 1]
- [ФАЙЛ 2, ЕСЛИ ЕСТЬ]

Шаблон Word:
- [ПУТЬ ИЛИ ФАЙЛ ШАБЛОНА]

Заполни только подтверждаемые данные.
Если для титульного листа, задания, аннотации или других сопроводительных документов не хватает реквизитов, сначала запроси их у меня и не выдумывай значения.
Если тип работы неоднозначен, сначала уточни, это СПО/дипломный проект или ВО/магистратура.
```

## 5. Минимальный Набор Реквизитов Для Запроса

Если агент готовит финальный документ, ему обычно нужно запросить:

- тип работы;
- уровень образования;
- точную тему;
- название вуза или колледжа;
- институт / факультет;
- кафедру / отделение;
- ФИО автора;
- ФИО руководителя;
- группу;
- город;
- год.
