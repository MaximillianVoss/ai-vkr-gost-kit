# AI VKR / DP GOST Kit

Самодостаточный комплект для ИИ-агента, который должен готовить и оформлять ВКР, дипломный проект или пояснительную записку по пользовательскому шаблону Word и ГОСТ с явным выбором профильной методички.

Состав:
- `word_agent_toolkit.py` - автономный CLI для работы с `.dotx/.dotm/.docx/.docm`: создание документа из шаблона, применение ГОСТ-профиля, обновление оглавления, экспорт PDF, добавление приложения с листингом и явный запуск VBA-макросов через Word COM.
- `tools/schemegen/` - встроенный генератор схем `JSON -> SVG` для блок-схем, `IDEF0`, `IDEF1X`, `IDEF3` и `UML class`, который можно использовать до вставки рисунков в пояснительную записку.
- `ai-client-vkr-gost-spec.json` - машиночитаемый профиль с маршрутами выбора между дипломным проектом СПО и ВКР магистратуры.
- `AGENT_PROMPTS_RU.md` - готовые prompt-шаблоны, которые можно отправлять другому ИИ-агенту вместе со ссылкой на репозиторий.
- `Профиль для ИИ агента - ВКР и ДП по ГОСТ.docx` - редактируемый Word-образец, который можно открыть, поправить и затем передать другому ИИ.
- `Профиль для ИИ агента - ВКР и ДП по ГОСТ.pdf` - PDF-предпросмотр того же образца.

Что требуется извне:
- локальный шаблон Word `.dotx` или `.dotm`, который не хранится в репозитории;
- при необходимости локальный PDF с методическими указаниями;
- Windows и установленный Microsoft Word для операций COM-автоматизации;
- Python с пакетами `python-docx` и `pywin32`.

Установка зависимостей:

```powershell
python -m pip install -r requirements.txt
```

Важно:
- пути к шаблону и методичке в JSON-спецификации являются машинно-зависимыми и должны быть заменены при переносе комплекта на другой компьютер;
- временные файлы рендера и проверки рекомендуется складывать в `tmp/docs/`;
- итоговый Word/PDF-профиль агента в этом комплекте считается каноническим;
- если пользователь написал только `ВКР`, агент не должен автоматически выбирать магистерский профиль: сначала нужно понять, это СПО/дипломный проект или ВО/магистратура.
- комплект является универсальным: в нем не должно быть жестко вшитых названий вузов, кафедр, ФИО, групп, городов и других персональных реквизитов.
- команда `inspect-template` теперь показывает тип Word-пакета и наличие `vbaProject.bin`, поэтому ею удобно сразу проверить, является ли шаблон macro-enabled.
- VBA-макросы не запускаются автоматически. Если нужно выполнить встроенные процедуры Word, их нужно явно передать через повторяемый флаг `--macro`.

Как работать с макросами:

- `.dotx` не хранит VBA-проект. Для встроенных макросов нужен `.dotm` или `.docm`.
- Команды `create-from-template`, `refresh-fields`, `export-pdf`, `finalize-doc` и `generate-agent-brief` принимают повторяемый флаг `--macro`.
- Макросы выполняются только по явному запросу через `Word.Application.Run`, а автоматические `AutoOpen/AutoExec` макросы принудительно блокируются.

Как использовать встроенную постобработку вместо VBA:

- В `finalize-doc` встроенная постобработка применяется автоматически: утилита нормализует специальные заголовки, форматирует обычный текст, приводит оглавление к ГОСТ-стилю, настраивает нумерацию страниц по секциям, выравнивает таблицы и центрирует встроенные рисунки без запуска VBA.
- В `refresh-fields` и `export-pdf` встроенную постобработку можно включить флагом `--builtin-layout`.
- Флаг `--insert-placeholder-captions` разрешает добавлять заглушки подписей для таблиц и рисунков, если подписи отсутствуют.
- Флаг `--strip-heading-numbering` снимает автоматическую нумерацию Word у заголовков ГОСТ, сохраняя сам текст заголовков.
- Для `ГОСТ Заголовок 1/2/3` по умолчанию задаются `20/18/16 pt`, полужирное начертание и интервалы `6 pt` до и после абзаца.
- Специальные разделы вроде `РЕФЕРАТ`, `ВВЕДЕНИЕ`, `ЗАКЛЮЧЕНИЕ`, `СПИСОК ИСПОЛЬЗОВАННЫХ ИСТОЧНИКОВ` и `ПРИЛОЖЕНИЕ` автоматически приводятся к регистру заголовка главы и остаются без нумерации.
- При экспорте PDF оглавление временно «замораживается» перед выгрузкой, чтобы в финальном PDF не оставалось ошибок вида `Ошибка! Закладка не определена.`.

Как генерировать тестовые ВКР:

- Команда `generate-sample-vkrs` создает набор синтетических ВКР на произвольные темы для проверки структуры документа, оглавления, списков и заголовков до уровня `1.1.1`.
- Генерация идет на основе пользовательского шаблона, поэтому итоговые документы наследуют стили `ГОСТ Текст`, `ГОСТ Заголовок 1`, `ГОСТ Заголовок 2` и `ГОСТ Заголовок 3`, если они есть в шаблоне.
- Тестовые образцы строятся с отдельными секциями для титульного листа, содержания и основного текста, поэтому номер страницы не печатается на титуле и содержании, но появляется с `Реферата` как страница `3`.
- По умолчанию документы складываются в папку `test`, а при флаге `--with-pdf` рядом создаются PDF-предпросмотры.

Как работать со схемами:

- если пользователю нужны блок-схемы, `IDEF0`, `IDEF1X`, `IDEF3` или `UML`-диаграммы для основной части или приложений, используй `tools/schemegen`;
- сначала сгенерируй или отредактируй структурированный JSON схемы, затем выполни `validate`, потом `render` или `render-all`;
- готовые `SVG` можно вставлять в Word как рисунки и дальше оформлять под подписи и ссылки через `word_agent_toolkit.py`;
- для `IDEF0` инструмент уже учитывает контекстную диаграмму, дочерние декомпозиции, `ICOM`-коды, grouped boundary arrows и каноническое ограничение `1` либо `3..6` блоков.

Быстрый старт для схем:

```powershell
Set-Location tools/schemegen
python -m schemegen validate .\examples\test_progression.json
python -m schemegen render-all .\examples\test_progression.json -o .\rendered\test_progression
```

Рекомендуемый общий pipeline:

1. Сначала определить профиль работы и шаблон Word.
2. Если в работе нужны схемы, подготовить их через `tools/schemegen`.
3. После этого собирать и нормализовать сам `.docx` через `word_agent_toolkit.py`.
4. В конце обновить поля, подписи рисунков, оглавление и экспортировать PDF.

Как выбирать методичку:

- `МУ к ДП.pdf`: использовать для дипломного проекта или ВКР СПО, когда работа относится к колледжу/техникуму и имеет четырехглавую прикладную структуру с обязательной экономической главой.
- `МУ ВКР.pdf`: использовать для ВКР магистратуры или другой ВКР высшего образования, когда обязательны реферат, анализ актуальности, формальное проектирование, технико-экономическое обоснование, анализ рисков и комплект сопроводительных документов.
- Если пользователь дал обе методички или сказал только `ВКР`, сначала определить уровень образования и только потом выбирать профиль оформления.

Как работать с реквизитами титульных и сопроводительных листов:

- в универсальных образцах оставлять плейсхолдеры вида `[НАЗВАНИЕ ВУЗА]`, `[ФИО АВТОРА]`, `[ФИО РУКОВОДИТЕЛЯ]`, `[КАФЕДРА / ОТДЕЛЕНИЕ]`, `[ГОРОД]`, `[ГОД]`;
- не копировать названия вузов, институтов, кафедр, ФИО и иные реквизиты из методичек как значения по умолчанию;
- если агент генерирует уже финальный документ на сдачу, он должен сначала запросить у пользователя недостающие реквизиты и только потом подставлять их вместо плейсхолдеров.

Быстрый старт:

```powershell
python word_agent_toolkit.py resolve-profile --work-type "дипломный проект" --education-level "СПО"
python word_agent_toolkit.py resolve-profile --work-type "магистерская ВКР" --education-level "магистратура"
python word_agent_toolkit.py inspect-template --template "C:\path\to\your-template.dotx"
python word_agent_toolkit.py generate-agent-brief --template "C:\path\to\your-template.dotx" --output "Профиль для ИИ агента - ВКР и ДП по ГОСТ.docx"
python word_agent_toolkit.py export-pdf --input "Профиль для ИИ агента - ВКР и ДП по ГОСТ.docx" --output "Профиль для ИИ агента - ВКР и ДП по ГОСТ.pdf"
python word_agent_toolkit.py finalize-doc --input tmp/docs/new-note.docx --macro "NormalizePageNumbers" --macro "AlignTablesAndFigures"
python word_agent_toolkit.py refresh-fields --input tmp/docs/new-note.docx --spec ai-client-vkr-gost-spec.json --builtin-layout
python word_agent_toolkit.py finalize-doc --input tmp/docs/new-note.docx --insert-placeholder-captions --strip-heading-numbering
python word_agent_toolkit.py generate-sample-vkrs --template "C:\path\to\your-template.dotx" --output-dir test --with-pdf
```
