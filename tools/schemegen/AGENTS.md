# Инструкции Для ИИ-Агентов В `tools/schemegen`

Используй этот инструмент, когда пользователю нужны:

- блок-схемы алгоритмов;
- `IDEF0`-контекст и декомпозиции;
- `IDEF1X`-схемы данных;
- `IDEF3`-сценарные диаграммы;
- `UML class`-диаграммы.

Рабочий порядок:

1. Сначала прочитай `README.md` в этой папке.
2. Определи тип диаграммы и собери ее в JSON.
3. Сначала запусти `python -m schemegen validate <json>`.
4. Затем запусти `python -m schemegen render ...` или `python -m schemegen render-all ...`.
5. Только после этого вставляй готовый `SVG` в Word-документ или приложение.

Правила:

- не рисуй схемы вручную через псевдографику или табличную разметку, если можно отрендерить их через этот инструмент;
- для `IDEF0` соблюдай правило: контекстная диаграмма содержит один блок, дочерняя диаграмма обычно содержит `3..6` блоков;
- если схема должна войти в пояснительную записку, сначала добейся структурной корректности JSON, а затем занимайся подписями и размещением рисунка в Word;
- если нужно показать длинные переходы в блок-схеме, используй `connector` и `offpage`, а не перегружай одну страницу лишними возвратными линиями.

Полезные файлы:

- `examples/test_progression.json` - основной набор тестовых схем;
- `examples/coursework_document.json` - пример комплекта схем под учебную работу;
- `tests/` - регрессионные проверки рендера и валидации.
