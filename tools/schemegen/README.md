# SchemeGen

`SchemeGen` - небольшая CLI-утилита на Python для генерации схем из JSON-документа. Она рассчитана на сценарий, где схему удобно сначала описать структурированно, а уже потом получить `SVG` для пояснительной записки, отчета или диплома.

Что уже поддерживается:

- блок-схемы (`flowchart`);
- `IDEF0`-диаграммы и их декомпозиции;
- `IDEF1X`-диаграммы данных для моделей БД;
- `IDEF3`-диаграммы сценариев с junction-узлами;
- `UML class`-диаграммы;
- соединители `connector` и `offpage` для длинных переходов в блок-схемах;
- автоматическая рамка/штамп для `IDEF0` в стиле старых CASE-инструментов;
- хранение нескольких диаграмм в одном JSON-документе;
- ссылки на детализацию через `decomposes_to`;
- выравнивание блоков по общей оси через одинаковые `row`/`column`, а также ручная доводка центров через `x`/`y`;
- каноническая раскладка `IDEF0`: один контекстный блок по центру, дочерние функции в неглубокой "лесенке";
- канонические ограничения `IDEF0`: `A-0` как один блок и дочерние диаграммы по правилу `3..6` блоков;
- маршрутизация стрелок по ортогональным коридорам с обходом блоков и верхним/нижним обходом для `IDEF`-feedback;
- `ICOM`-коды и туннели на граничных стрелках `IDEF0`;
- автоматический `fan-out/fan-in` для одинаковых граничных интерфейсов `IDEF0`: одна магистраль и ветви к функциям;
- для блок-схем loopback-стрелки в циклах теперь по умолчанию уводятся на внешний rail, а не режут основной поток;
- единый стиль диаграммы, узлов и стрелок: цвет, толщина, заливка, радиус, шрифт, пунктир, подложка подписи;
- валидация формата перед рендерингом;
- рендер в `SVG` без внешних GUI-зависимостей.

## Установка

```powershell
python -m pip install -e .
```

Либо можно запускать без установки:

```powershell
python -m schemegen --help
```

## Команды

Проверить документ:

```powershell
python -m schemegen validate .\examples\coursework_document.json
```

Сгенерировать одну схему:

```powershell
python -m schemegen render .\examples\flowchart_only.json -o .\out\flowchart.svg
```

Сгенерировать все схемы из одного документа:

```powershell
python -m schemegen render-all .\examples\coursework_document.json -o .\out
```

Получить JSON Schema:

```powershell
python -m schemegen schema
```

Посмотреть встроенные примеры:

```powershell
python -m schemegen examples
```

## Использование из ИИ-агента

Самый удобный путь для агента:

1. Сгенерировать JSON по описанию пользователя.
2. Прогнать `validate`.
3. Отрендерить `SVG`.

Можно не создавать промежуточный файл, а передавать JSON через `stdin`:

```powershell
Get-Content .\examples\flowchart_only.json | python -m schemegen render --stdin -o .\out\stdin-flow.svg
```

## Формат документа

Внутренний формат хранит сразу весь набор схем:

```json
{
  "schema_version": "1.0",
  "title": "Набор схем",
  "diagrams": [
    {
      "id": "algorithm_main",
      "title": "Основной алгоритм",
      "type": "flowchart",
      "nodes": [
        {
          "id": "start",
          "label": "Начало",
          "kind": "start",
          "row": 0,
          "column": 0
        }
      ],
      "edges": []
    }
  ]
}
```

### Узлы

Общие поля узла:

- `id`: уникальный идентификатор;
- `label`: текст внутри блока;
- `kind`: тип блока;
- для `flowchart` можно использовать, помимо базовых блоков, еще `connector` и `offpage`;
- `row`, `column`: позиция на логической сетке;
- `x`, `y`: опциональные координаты центра блока для ручного выравнивания;
- `width`, `height`: опциональные размеры;
- `code`: код функции для `IDEF0`, например `A0`, `A1`;
- `decomposes_to`: идентификатор диаграммы-детализации;
- `stereotype`: стереотип для `UML class`;
- `attributes`: список атрибутов для `UML class`;
- `operations`: список операций для `UML class`;
- `columns`: список колонок для `IDEF1X`;
- `style`: локальное переопределение оформления узла.

### Ребра

Общие поля ребра:

- `id`: уникальный идентификатор;
- `source`, `target`: идентификаторы узлов;
- `label`: подпись на стрелке;
- `kind`: тип связи для `UML class`/`IDEF1X`;
- `icom`: код граничной стрелки `IDEF0` (`I1`, `C2`, `O1`, `M3`);
- `source_side`, `target_side`: сторона подключения (`top`, `right`, `bottom`, `left`);
- `tunnel_source`, `tunnel_target`: пометка туннеля на соответствующем конце стрелки `IDEF0`;
- `role`: роль `IDEF0` (`input`, `control`, `output`, `mechanism`);
- `source_label`, `target_label`: подписи у концов связи, например кратности;
- `style`: локальное переопределение оформления стрелки.

Для `IDEF3` используются обычные стрелки между `uob`-блоками и junction-узлами (`junction_x`, `junction_and`, `junction_or`).

Для `UML class` поддерживаются связи `association`, `inheritance`, `aggregation`, `composition`, `dependency`, `realization`.

Для `IDEF1X` поддерживаются связи `identifying` и `non_identifying`.

Для `IDEF0` можно задавать внешние стрелки через `null` на одном из концов. Например:

```json
{
  "id": "control_rules",
  "source": null,
  "target": "manage_requests",
  "role": "control",
  "icom": "C1",
  "label": "Регламент"
}
```

Для дочерних `IDEF0`-диаграмм у граничной стрелки должен быть либо `icom`, либо туннель на незамкнутом конце. Например:

```json
{
  "id": "storage_hint",
  "source": null,
  "target": "assemble_report",
  "role": "mechanism",
  "label": "Архив шаблонов",
  "tunnel_source": true
}
```

### Рамка IDEF0

Для диаграмм `idef0` рамка и штамп включаются автоматически. При необходимости поля можно переопределить через `frame`:

```json
{
  "frame": {
    "author": "ИИ-агент",
    "project": "Дипломный проект",
    "date": "13.04.2026",
    "revision": "1",
    "status": "Черновик",
    "paper_fill": "#ffffff",
    "reader": "Руководитель",
    "context": "ВЕРХ",
    "page": "1",
    "node_ref": "A0"
  }
}
```

Статусы в штампе лучше использовать из классического набора: `Рабочий проект`, `Черновик`, `Рекомендовано`, `Публикация`.

### Стили

У диаграммы можно задать общую тему:

```json
{
  "style": {
    "background_fill": "#f8fafc",
    "node": {
      "fill": "#ffffff",
      "stroke": "#334155",
      "stroke_width": 1.6,
      "corner_radius": 8,
      "font_size": 13
    },
    "edge": {
      "stroke": "#1d4ed8",
      "stroke_width": 1.5,
      "text_color": "#1e3a8a",
      "dasharray": "7 4",
      "label_fill": "#dbeafe"
    }
  }
}
```

Те же поля можно переопределять у конкретного узла или конкретной стрелки через их собственный `style`.

## Практический смысл для курсовых и диплома

Этот формат удобен тем, что:

- схема хранится не только как картинка, но и как восстанавливаемая структура;
- ИИ-агент может генерировать схему по текстовому описанию без OCR;
- одну и ту же логику можно экспортировать в разные схемы и версии;
- декомпозицию `IDEF0` можно хранить в одном файле вместе с контекстной диаграммой.

Если дальше понадобится, поверх этого каркаса можно добавить:

- экспорт в `PNG` через конвертацию из `SVG`;
- экспорт в `draw.io` или Mermaid;
- генерацию JSON по тексту через локальную LLM/API;
- более строгие правила оформления под конкретную кафедру или ГОСТ-шаблон.
